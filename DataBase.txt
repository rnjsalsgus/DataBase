DB(DataBase)
- 데이터베이스는 구조화된 정보 또는 데이터의 조직화된 모음

DBMS(DataBase Management System)
- 데이터베이스와 최종 사용자 또는 프로그램 간의 인터페이스 역할을 하여 사용자가 정보의 구성 및 최적화 방법을 검색, 업데이트 및 관리할 수 있도록 해주는 소프트웨어

SQL
- 데이터를 쿼리, 조작 및 정의하고 액세스제어를 제공하기 위해 거의 모든 관계형 데이터베이스에서 사용되는 프로그래밍 언어


관계형 데이터베이스

- 고객 정보를 저장하는 공간(회원가입시 데이터삽입)
ex) 이름, 전화번호, e-mail 등...
- 고객 주문을 저장하는 공간(회원이 주문시 데이터삽입)
ex) 주문상품, 금액 등...

1. 데이터를 중복하여 저장하지 않는다. 두 데이터베이스를 연결(관계형 데이터베이스)
-> 중복하여 저장하지 않기 때문에 데이터의 신뢰성이 높다. 
2. 데이터 저장 규칙이 엄격하다.
3. 복잡한 트랜젝션에 적합하다.
-> 트랜젝션 : 데이터 관련 논리적인 일의 최소 단위
ex) instagram 좋아요 누르기
    -> 좋아요 숫자가 늘어남, 사용자 하트가 색이 바뀜.
    -> 둘 다 성공하면 OK(커밋)
    -> 둘 중 하나라도 실패하면 NO(롤백)
4. 전자상거래(이커머스) 플랫폼에 적합하다.
5. MySQL, MariaDB, PostgreSQL 등등...

비관계형 데이터베이스

1. 데이터를 중복하여 저장해도 상관없다.
2. 데이터 저장 규칙이 느슨하다.
3. 빅데이터 분석에 적합하다.
4. 소셜 미디어 플랫폼에 좋다.
5. MongoDB ...

MySQL을 많이 사용하는 이유
1. 사용하기가 쉽다.
2. 보안을 최우선으로 한다.
3. 오픈소스 (라이센스 비용이 없다.)
4. 강력한 트랜젝션 제공

서버(데이터를 가지고 있는 제공자)
클라이언트(데이터를 요구하는 사용자)
